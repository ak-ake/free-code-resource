CSS Responsive Design

Create websites that look great on all devices using responsive design principles.

Mobile-First Approach

Start with mobile design and enhance for larger screens.

/* Mobile styles (base) */ .container { width: 100%; padding: 1rem; } /* Tablet and above */ @media (min-width: 768px) { .container { width: 750px; margin: 0 auto; } } /* Desktop and above */ @media (min-width: 1024px) { .container { width: 970px; } }

Responsive Breakpoints

  • Mobile - 320px to 767px
  • Tablet - 768px to 1023px
  • Desktop - 1024px and above
  • Large Desktop - 1440px and above

Flexible Grid System

.grid { display: grid; grid-template-columns: 1fr; gap: 1rem; } @media (min-width: 768px) { .grid { grid-template-columns: 1fr 1fr; } } @media (min-width: 1024px) { .grid { grid-template-columns: 1fr 1fr 1fr; } }

Responsive Navbar

nav { display: flex; flex-direction: column; } nav a { padding: 1rem; } @media (min-width: 768px) { nav { flex-direction: row; gap: 2rem; } nav a { padding: 0; } }

Responsive Units

  • px - Fixed pixels
  • em - Relative to parent font size
  • rem - Relative to root font size
  • % - Percentage of parent
  • vw - Viewport width
  • vh - Viewport height
  • vmin - Smaller of vw or vh
  • vmax - Larger of vw or vh

Responsive Images

img { max-width: 100%; height: auto; } .responsive-img { width: 100%; display: block; }

Media Query Features

  • min-width - Minimum viewport width
  • max-width - Maximum viewport width
  • orientation - Portrait or landscape
  • prefers-reduced-motion - Accessibility preference
  • prefers-color-scheme - Light or dark mode

Dark Mode Support

/* Light mode - default */ body { background: white; color: black; } /* Dark mode */ @media (prefers-color-scheme: dark) { body { background: #1a1a1a; color: #ffffff; } }
Best Practices:
  • Always use mobile-first approach
  • Test on real devices
  • Use flexible units (em, rem, %)
  • Don't rely on screen size alone
  • Optimize images for different sizes

Use Cases

  • Creating mobile-first websites
  • Building adaptive layouts
  • Supporting multiple devices
  • Improving accessibility
  • Creating progressive web apps
