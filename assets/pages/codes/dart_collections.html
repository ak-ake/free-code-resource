Dart Collections

Work with Lists, Sets, Maps, and other collections.

Lists

// Create lists List<String> names = ["Alice", "Bob"]; var numbers = <int>[1, 2, 3]; // List operations names.add("Charlie"); names.remove("Bob"); names.forEach((name) => print(name)); // List methods names.length; names.isEmpty; names.map((n) => n.toUpperCase());

Maps

// Create maps Map<String, int> scores = { "Alice": 95, "Bob": 87 }; // Map operations scores["Charlie"] = 92; scores.remove("Bob"); scores.keys; scores.values; // Iteration scores.forEach((name, score) { print("$name: $score"); });

Sets

// Create sets Set<String> colors = {"red", "green", "blue"}; // Set operations colors.add("yellow"); colors.remove("red"); colors.contains("blue"); // => true // Set algebra var union = colors.union({"purple"}); var intersection = colors.intersection({"green"});
  • List - Ordered, mutable collection
  • Map - Key-value pairs
  • Set - Unique elements
  • Iterable - Base for all collections
  • Queue - FIFO operations
Best Practices:
  • Use type annotations for clarity
  • Prefer List/Map/Set for common cases
  • Use spread operator for combining collections
  • Leverage functional methods like map, filter

Use Cases

  • Data storage
  • Model properties
  • Configuration objects
  • Filtering and transforming data
