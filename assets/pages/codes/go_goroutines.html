Go Goroutines

Leverage Go's lightweight concurrency model.

Basic Goroutines

package main import "fmt" func sayHello(name string) { fmt.Println("Hello", name) } func main() { go sayHello("Alice") go sayHello("Bob") time.Sleep(1 * time.Second) }

Goroutines with WaitGroup

package main import ( "fmt" "sync" ) func main() { var wg sync.WaitGroup for i := 0; i < 3; i++ { wg.Add(1) go func(n int) { defer wg.Done() fmt.Println(n) }(i) } wg.Wait() }

Spawning Multiple Goroutines

package main import ( "fmt" "sync" ) func worker(id int) { fmt.Printf("Worker %d working\n", id) } func main() { var wg sync.WaitGroup for i := 1; i <= 5; i++ { wg.Add(1) go func(n int) { defer wg.Done() worker(n) }(i) } wg.Wait() }
  • go keyword - Launch goroutine
  • WaitGroup - Wait for goroutines
  • Lightweight - Thousands of goroutines
  • Scheduled - OS thread agnostic
Best Practices:
  • Always use WaitGroup to synchronize
  • Avoid goroutine leaks
  • Use channels for communication
  • Be careful with shared state

Use Cases

  • Concurrent tasks
  • Web servers
  • Parallel processing
  • Background jobs
