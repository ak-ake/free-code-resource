HTML Semantic Elements

Master semantic HTML to improve accessibility, SEO, and code structure.

What are Semantic Elements?

Semantic elements clearly describe their meaning to both the browser and developer. They improve accessibility and SEO by providing context about the content.

Key Semantic Elements

Document Structure

<header> - Page or section header </header> <nav> - Navigation links </nav> <main> - Main content </main> <article> - Independent content </article> <section> - Grouped content </section> <aside> - Sidebar or supplementary content </aside> <footer> - Page or section footer </footer>

Complete Semantic Document Structure

<!DOCTYPE html> <html> <head> <title>Blog Post</title> </head> <body> <header> <h1>My Blog</h1> <nav> <ul> <li><a href="/">Home</a></li> <li><a href="/posts">Posts</a></li> </ul> </nav> </header> <main> <article> <h2>Learning Semantic HTML</h2> <p>Semantic HTML makes your code more meaningful...</p> </article> <aside> <h3>Related Articles</h3> </aside> </main> <footer> <p>© 2024 My Blog</p> </footer> </body> </html>

Text Semantic Elements

<strong>Important text</strong> - Strong importance <em>Emphasized text</em> - Emphasis <mark>Highlighted text</mark> - Marked/highlighted <time>2024-01-15</time> - Date/time <address>Contact info</address> - Contact information <code>var x = 5;</code> - Code snippet <figure>...</figure> - Self-contained illustration <figcaption>Caption</figcaption> - Figure caption

Benefits of Semantic HTML

  • Accessibility - Screen readers interpret content correctly
  • SEO - Search engines understand content structure
  • Maintainability - Code is easier to understand and modify
  • Better Browser Support - Consistent rendering across browsers
  • Mobile Friendly - Simpler responsive design implementation
Pro Tips:
  • Use one <main> per page for primary content
  • Use <nav> for major navigation links
  • Use <article> for self-contained content
  • Use <section> to group thematically related content

Use Cases

  • Building accessible websites
  • Improving SEO rankings
  • Creating blog posts and articles
  • Building news websites
  • Documentation sites
  • Content management systems
