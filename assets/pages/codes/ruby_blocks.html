Ruby Blocks and Procs

Master Ruby's powerful block and proc mechanisms.

Blocks

# Iterate with blocks [1, 2, 3].each { |n| puts n * 2 } # Method with yield def greet puts "Hello" yield puts "Goodbye" end greet { puts "Nice to meet you" }

Procs

# Create a proc multiply = Proc.new { |x| x * 2 } puts multiply.call(5) # => 10 # Pass proc as argument def process(&block) block.call("data") end process { |data| puts data }

Lambdas

# Lambda square = ->(x) { x * x } puts square.call(4) # => 16 # Difference: lambdas check arity my_lambda = ->(x) { puts x } my_lambda.call(1, 2) # => Error!
  • Blocks - Unnamed functions passed to methods
  • Procs - Object versions of blocks
  • Lambdas - Strict procs with arity checking
  • yield - Call blocks from methods
Best Practices:
  • Use blocks for simple operations
  • Use lambdas for strict validation
  • Prefer lambdas over procs
  • Handle block parameters explicitly
