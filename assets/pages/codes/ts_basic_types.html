TypeScript Basic Types

Understanding TypeScript's type system and basic types.

Primitive Types

// String let name: string = "Alice"; // Number let age: number = 25; // Boolean let isActive: boolean = true; // Any (avoid using) let data: any = "anything"; // Union Types let id: string | number = 123;

Arrays and Tuples

// Array of strings let colors: string[] = ["red", "blue"]; // Array generic syntax let numbers: Array<number> = [1, 2, 3]; // Tuple let tuple: [string, number] = ["test", 42];

Enums

enum Direction { Up = 1, Down = 2, Left = 3, Right = 4 } let dir: Direction = Direction.Up;
  • string - Text values
  • number - Integers and floats
  • boolean - true/false values
  • null & undefined - Absence of value
  • any - Dynamic typing (use cautiously)
  • void - Function return type
Best Practices:
  • Always specify explicit types
  • Avoid using `any` type
  • Use union types for multiple possibilities
  • Use enums for fixed sets of values

Use Cases

  • Type safety in large projects
  • IDE autocompletion
  • Catching errors early
  • Self-documenting code
