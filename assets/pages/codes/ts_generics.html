TypeScript Generics

Create reusable components that work with multiple types.

Generic Functions

function identity<T>(arg: T): T { return arg; } let output1 = identity<string>("hello"); let output2 = identity<number>(42);

Generic Classes

class Box<T> { contents: T; constructor(value: T) { this.contents = value; } getValue(): T { return this.contents; } } const box = new Box<string>("test");

Generic Constraints

interface HasLength { length: number; } function printLength<T extends HasLength>(arg: T) { console.log(arg.length); } printLength("hello");

Generic Interfaces

interface Pair<T, U> { first: T; second: U; } const pair: Pair<string, number> = { first: "age", second: 25 };
  • Generic type parameters - <T>, <T, U>
  • Type constraints - extends keyword
  • Default types - T = string
  • Keyof operator - Get object keys
Best Practices:
  • Use meaningful generic names (T, U, K, V)
  • Add constraints for better type safety
  • Use generics for reusable components
  • Avoid unnecessary generic parameters

Use Cases

  • API utilities
  • Data structures
  • React components
  • State management
