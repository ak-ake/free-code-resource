TypeScript Interfaces

Define contracts for object shapes and structure.

Basic Interface

interface User { id: number; name: string; email: string; active?: boolean; } const user: User = { id: 1, name: "John", email: "john@example.com" };

Interface Methods

interface Database { query(sql: string): any[]; connect(): void; disconnect(): void; } class PostgreSQL implements Database { query(sql: string) { } connect() { } disconnect() { } }

Interface Inheritance

interface Person { name: string; } interface Employee extends Person { id: number; salary: number; }
  • Property definitions - Define object structure
  • Optional properties - Properties marked with ?
  • Method signatures - Define function contracts
  • Inheritance - Extend other interfaces
  • Readonly properties - Immutable properties
Best Practices:
  • Use interfaces for object contracts
  • Make properties optional when needed with ?
  • Extend interfaces for code reuse
  • Use readonly for immutable properties

Use Cases

  • API response types
  • Component props
  • Database models
  • Configuration objects
